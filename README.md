# JavaNovice

JavaでCLI（コマンドラインインターフェース）アプリケーションを開発・実行するためのシンプルな学習用環境です。
外部の複雑なビルドツール（MavenやGradleなど）を使用せず、Java標準のコンパイラ（`javac`）とシェルスクリプトを組み合わせた、直感的で軽量な構成になっています。

## ディレクトリ構成

```text
JavaNovice/
├── README.md           # 本ファイル
├── run.sh              # ビルド＆実行用シェルスクリプト
├── src/                # Javaソースコード用のディレクトリ
│   ├── Main.java       # デフォルトのCLIプログラム（サンプル）
│   └── Test001.java    # 個別実行検証用のプログラム
└── .vscode/
    └── launch.json     # VS Codeデバッグ実行設定
```

## 新しいJavaプログラムの追加手順

新しく別のJavaプログラムを作成して実行したい場合は、以下の手順で行います。

1. **Javaファイルの作成**
   `src/` ディレクトリの中に、新規のJavaファイル（例: `Test002.java`）を作成します。
   ファイル内に `main` メソッドを実装します。

   ```java
   // src/Test002.java
   public class Test002 {
       public static void main(String[] args) {
           System.out.println("こんにちは、新規プログラムです！");
       }
   }
   ```

2. **プログラムの実行**
   `run.sh` の第1引数にクラス名（またはファイルパス）を指定して実行します。
   ```bash
   ./run.sh Test002
   ```
   スクリプトが自動的に変更を検知してコンパイルを行い、実行結果を出力します。

## run.shの使い方

`run.sh` は、Javaプログラムのコンパイル（`javac`）と実行（`java`）を1ステップで行うスクリプトです。

### 1. デフォルトプログラム（Main.java）を実行する

ターゲットを省略して実行すると、デフォルトで `src/Main.java` が動作します。

```bash
./run.sh [オプション]
```

#### Main.java がサポートするオプション一覧:
- `-h, --help`         : ヘルプメッセージを表示して終了します。
- `-v, --version`      : バージョン情報を表示して終了します。
- `-n, --name <名前>`   : 挨拶する対象の名前を指定します。
- `-i, --interactive`  : キーボード入力を行う「対話モード」を開始します。

### 2. 個別指定したJavaファイルを実行する

実行したいファイル名（拡張子なしのクラス名、またはファイルパス）を第1引数に指定します。プログラムへの引数（オプション）はその後ろに続けます。

```bash
./run.sh <クラス名またはファイルパス> [プログラムへの引数...]
```

**実行例:**
```bash
# クラス名で指定する場合
./run.sh Test001 arg1 arg2

# ファイルパスで指定する場合
./run.sh src/Test001.java arg1 arg2
```

## VS Codeでのデバッグ

VS Codeで開発する場合、`F5` キーを押すか、「実行とデバッグ」メニューから以下の構成を選択してデバッグ実行が可能です。

- **Java CLI - 通常起動**: `Main.java` を引数なしで実行します。
- **Java CLI - 引数付き起動**: `Main.java` に `--name Taro` を指定して実行します。
- **Java CLI - 対話モード起動**: `Main.java` を対話モードで実行します。
